Query processing in ProLinks is powered by Solr. Therefore complex queries can be formulated, which include the use of wildcards, logic operations, and special characters.


Example 1: Default mode.
The default behavior is to combine all keywords using AND logic. For example, to find all instances of carboxylases enzymes in the set of Escherichia genomes available in ProLinks, we simply type the query:

carboxylases escherichia


Example 2: logical operators and special characters.
Find all proteins in E. coli K12 MG1655 whose name contains the letters 'hist'. Since there are several E. coli genomes already sequenced, we will use the substrain to identify the genome:

hist* AND MG1655

Notice that the logical operators must be written with capital letters (i.e. AND, OR, NOT). Alternatively we can force both terms in the results by using the plus sign as prefix:

+hist* +MG1655

If you want to discard kinases from the results, the following two queries are equivalent:

hist* AND MG1655 NOT kinase

+hist* +MG1655 -kinase

You will seldom have to use the plus sign, if at all, since by default all keywords typed should be in the results. So these queries can be simplified to:

hist* MG1655 -kinase


Example 3: field names, parenthesis and phrases.
Solr uses field names to index information (e.g. label, taxid, xref, taxon, etc.) To search for all his and pro genes in E. coli K12 MG1655 we can use the following queries:

(label:his? OR label:pro?) taxon:MG1655

(label:his? OR label:pro?) taxid:511145

label:(his? OR pro?) taxon:MG1655

Phrases must be quoted:

label:(his* OR label:pro*) taxon:"substr MG1655"

"DNA-binding response regulator" Escherichia
