Submit a protein sequence      

This tutorial is divided in 4 sections:
  • The input form: General description of the input form.

  • Search parameters: Description of the available parameters to control the Blast search.

  • Blast results: Description of how the blast output is presented to the user.

The input form

If the protein you are working with belongs to a genome that is not fully sequenced yet, or your genome is not included in the set of genomes for which ProLinks has inferences, you can still submit the amino acid sequence of your protein of interest. Your sequence will be blasted against all proteins represented in ProLinks and the list of best hits will be reported back to you. If one of the matches belongs to a protein you consider to be significantly related to your protein, you'll have the opportunity to submit that protein to search for functional linkages.

The input form for sequence submition is presnted in Figure 1. You can either copy/paste your sequence in the form, or upload a file with your sequence in fasta format.


Figure 1. Input form. Two options are avaialble to to submit a protein sequence: a) copy/paste the sequence, and b)upload a file with the sequence. The question marks next to each text field and search parameter open a window with the respective help for that field.


If you click on the link (in fasta format) on top of the input text fields. A window will open explaining the fasta format and an example sequence will be shown. Copy that sequence and paste it on the input box.


Blast search parameters

The parameters to control de blast search are shown in Figure 2. The option Matches per page indicates how many significant blast matches will be presented to the user once Blast finishes comparing the input sequence against all proteins in ProLinks. The parameter E-value indicates the maximum E-value above which blast hits are not considered significant. % of identities in Query reflects the minimum percentage of residues in the query protein that must be identical to the residues in the target protein. Normally this value should be close to one, indicating that most residues in the query were related to the target protein, thus increasing the probability that both proteins carry out the same function. % of identities in Target indicates the minimum percentage of residues in the target protein that should be identical to the query protein. This value can be lower than 50% to account for the possibility that the target is much bigger than the query, and that the target could actually be a fused protein. But depending on the query protein you may be intrested in proteins that are more or less the same size as the query.

Figure 2. Search Parameters tab. Available options are described above. Also, if you click on the question marks next to each field's description a pop-up window will open with the corresponding explanation.




Blast matches

Once you click on the Blast Sequence button. The sequence is submitted to our cluster to be blasted against all proteins in ProLinks. Figure 3, illustrates how ProLinks presents the top blast matches for the query.

Figure 3. Top blast matches for query sequence. Blast results are presented in a table with the main columns: E-value, Query identities, Target identities, and Alignment identities. If you click on the header of any column a window will open with an explanation of the the values in that column. The whole table can be sorted by the values in any of the columns (click on the figure to see the full size image).

Once you select the best match for your query you can submit the matched protein to ProLinks. In this case the first match produced 100% identities in the aligniment, query, and matched proteins, which indicates that the query Protein is actually contained in ProLinks and it only identified itself by running blast.

To submit the protein directly to ProLinks using default parameters click on the icon. Alternatively, if you want to control the parameters of the submission, click on the icon.


   Page navigation

If your query returned more than one page of blast matches, you will be able to browse through pages of blast matches just like if you were navigating through pages of inferred linkages. Therefore, you can browse through the matches by clicking in the arrows in the footer of the results table. Where > takes you to the next page, < retrieves the previous page, >> sends you to the last page, << takes you back to the first page, and all presents all paths found in one single page.
