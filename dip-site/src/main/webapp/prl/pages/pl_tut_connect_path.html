The input form

The input form to search for paths of linakges connecting two protein identifiers is presented in Figure 1. The identifiers must be unique or ProLinks will throw an exception.


Figure 1. Input form. The user must provide the protein IDs that ProLinks will attempt to connect with functional linkages. The figure shows the example IDs that will be submitted. By clicking on any question mark, a popup window will open with an explanation of the corresponding topic. Parameters Confidence value, Network distance, and Identify links with were explained in a in another tutorial. The format options Connecting paths and Paths per page are explained below (click on the figure to see the full size image).


Option Connecting paths determines the paths that will be returned by ProLinks, which can be the shortest, longest or all. Finally, Paths per page works similarly to the option Linkages per page, which has been already discussed. The difference is that here the unit of display is not a single linkage, but the whole path of linkages that connects both query proteins. Therefore a page displaying 5 paths may contain up to 20 functional linkages if the Network Distance parameter is set to 4.

   Page navigation

The procedure to browse through pages of paths is the same as browsing through pages of inferred linkages. Therefore, you can browse through the inferred paths by clicking in the arrows in the footer of the results table. Where > takes you to the next page, < retrieves the previous page, >> sends you to the last page, << takes you back to the first page, and all presents all paths found in one single page.
