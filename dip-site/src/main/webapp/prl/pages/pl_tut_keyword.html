The input form

The input forms for all advanced queries are organized in three main sections: User input, search parameters and ouput format options (see Figure 1). You can submit a a protein identifier or keywords as explained in the corresponding tutorial. You can also Click here for tips for a few example queries.


Figure 1. Input form. The user must provide keywords related to the protein of interest. The figure shows a default example with the locus tag that we will submit to ProLinks. By clicking on any question mark, a popup window will open with an explanation of the corresponding topic.


   Search Parameters

There are four parameters to control the search for linkages and how results will be presented:

  • Confidence value. Given that all inference methods differ in their underlying assumptions and statistical approaches their respective scores (i.e. P-values) are not comparable. We created a new standarized metric to allow comparisons between methods, and to quantify our global confidence that a given inference is a true positive. We call this score the ProLinks Confidence Value, and it is derived from the recovery rate of KEGG pathways, DIP core interactions, and known operons. For each method, we calculate the probability that inferred linkages are true positives given their recovery rates and their P-values. We recomend starting your search with 0.6 and depending on the number of linkages returned, work your way down to 0.4.

  • Network Distance. This refers to how deep in the network of functional associations we are going to search for proteins linked to the query protein. For example, at depth 1 we are looking at direct links (e.g. A→B) and for depth > 1 we find increasing levels of indirect links (e.g. A→b→C, A→b→c→D, etc.)

  • Identify links with. The tag ProLinks will use to identify each linked protein when displaying the inferred linkages.

  • Linkages per page. Number of links that will be displayed per page when ProLinks returns the inferences for the query protein.



Inferences of functional linkages

Let's use the parameters shown in Figure 1, and search for linkages up to 2 edges away from the query (Network distance = 2). We obtain the inferences presented in Figure 2.

   Page navigation

When the number of inferred linkages is greater than the number specified in the input form (see Figure 1), the results are paginated. In both cases, Figure 2 and Figure 3 return more than 10 linkages. You can browse through the inferred functional linkages by clicking in the arrows in the footer of the results table. The arrow > takes you to the next page, < retrieves the previous page, >> sends you to the last page, << takes you back to the first page, and if present, the all link will show all linkages found in one single page.



The network of functional interactions

During the process of interpreting functional linkages, it may be helpful to visualize the linkages as a network of interactions. To achieve this, we use the MiSink plugin (see the article for details) to submit the inferred linkages to Cytoscape (an interactive tool for visualizing biological networks). The following icon is located at the top right corner of the results table. By clicking on the icon the linkages in the current table of inferences is submitted to Cytoscape.


In order to be able to successfully visualize the network, it is MANDATORY that both MiSink and Cytoscape are installed and running at the time you click on the icon. If Cytoscape is running and MiSink is properly installed, by clicking on the network icon in Figure 2 and Figure 3, the inferred linkages in each table will be sent to Cytoscape.

When you click on the network icon, only the current page of linkages are sent to Cytoscape. That is why you need to display all linkages in one page if you want Cytoscape to plot the network including all inferred linkages. Figure 4 and Figure 5 show the corresponding networks.
